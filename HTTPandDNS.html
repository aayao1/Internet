What is DNS (Domain Name System)?

The Domain Name System is like the phonebook of the internet. It converts human-readable domain names like example.com into IP addresses like 192.158.1.3. Computers communicate using IP addresses, but humans use domain names because they are easier to remember.

An important characteristic to note about DNS is that it's an open public communication protocol - for schools, the government, etc. This makes such an open network susceptible to cyber attacks such as DNS spoofing. (Safe websites prevent this by asking your web browser to communicate on a secure channel, using SSL and TLS.)

Whenever you type a URL in your browser, your computer asks the DNS server. When asked, the server is responsible for finding the correct IP address of that website - whether that DNS knows the address already, or has to ask other DNS servers.

How DNS servers work together:

  1. DNS servers are connected in a distributed hierarchy. The DNS hierarchy starts with ROOT DNS SERVERS at the highest level. There are 13 root server clusters worldwide, but they do not store the complete DNS records.

  2. Instead, they provide pointers to the next level in the hierarchy - TOP-LEVEL DOMAIN SERVERS (TLD name servers).

  3. When your local DNS server asks the root server, it gets directed to the appropriate TLD name server (.org, .com, .net, etc.).

What is HTTP (Hypertext Transfer Protocol)?

HTTP stands for Hypertext Transfer Protocol. It's a protocol - which is an agreed upon standard for how data is sent around the internet. In this case, it is agreed that HTTP is in charge of transmitting web pages over the Internet.

More detailed explanation of web browsing:

  1. User enters a URL (Uniform Resource Locator) into the web browser's address bar. A URL consists of a protocol (http:// or https://), the domain name (www.example.com), and sometimes a path (e.g., /about).

  2. Follows DNS protocol by translating the domain name into a machine-readable IP address (discussed in earlier section).

  3. Establishing a TCP Connection with the web server.

  4. HTTP request. Once the TCP connection is established, the browser sends a HTTP request (commonly GET or POST).

  5. HTTP response. The server processes the HTTP request and responds with an HTTP response.

Frequently Asked Questions

Q: What is a cookie in HTTP?

A: A cookie is a small piece of data similar to an ID# stored by the browser and sent along with HTTP requests to the server. It's what websites use to remember who you are.


Q: What is the difference between HTTP, HTTPS, and HTML?

A: HTTP is a protocol used to transfer data across the web, but it's not secure. HTTPS is a secure version of HTTP - it ensures HTTP requests are secure and protected using encryption (SSL/TLS). HTML is the language used to tell a web browser how to make a page look. Thus, while HTTP/HTTPS handles how web data is transmitted (securely or not), HTML deals with what the data (the webpage) actually looks like and how it's structured.
